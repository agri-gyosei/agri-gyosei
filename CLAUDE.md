# プロジェクト総合指示書

## オーナーの想いと使命

オーナーは、経営企画10年・Webプロデューサー・ITコンサルタントを経て、現在某企業のSEO/AI担当として働きながら、行政書士試験合格を目指している（2025年11月受験・2026年1月開業予定）。

**核心にある想い：**
日本の農地が海外資本に買いたたかれ、気づけば日本の国土が外国人所有になる事態を防ぎたい。ロシアのダーチャ文化を参考に、日本国民が楽しみながら農業・自給自足を行い、日本の国土を日本人の手に守り続けたい。日本国民の資産・労働力・金銭が海外流出し続けて貧しくなるのを止めたい。

**ただし**：この想いをそのまま伝えても響かない。だから「兼業農家という楽しい選択肢」を切り口にする。

## サイト構成

### agri-gyosei.com/sikaku（行政書士サイト）
- ターゲット：行政書士試験を半年で合格したい社会人
- コンセプト：最短合格のための実践的情報発信
- 記事生成：`lib/agents/content-agent.ts` → `articles`テーブル
- 毎朝5:00 Vercel Cronで自動生成

### agri-gyosei.com/dacha（ダーチャサイト）
- ターゲット：兼業農家に興味がある都市住民・サラリーマン
- コンセプト：楽しい自給自足・日本版ダーチャという生き方
- キーワード：ダーチャ・ホルムズ海峡封鎖・自給自足・国土を守る・日本人総農家・畑付き別荘
- 記事生成：`lib/agents/dacha-agent.ts` → `dacha_articles`テーブル
- 毎朝5:30 Vercel Cronで自動生成

## AIエージェントの役割分担

| エージェント | ファイル | 役割 |
|------------|---------|------|
| 行政書士ライター | content-agent.ts | 行政書士試験記事の生成 |
| ダーチャライター | dacha-agent.ts | 兼業農家・ダーチャ記事の生成 |
| ファクトチェッカー | fix-article-factcheck.ts | 法的・事実確認 |
| エディター | patch-article.ts | 記事修正・品質管理 |

## 最重要ルール

1. **ファクトチェックは必須**：法律・数値・URLは必ず正確に。行政書士として信頼が命綱
2. **URLリンクはURLのみをhrefに**：前後の日本語テキストをリンクに含めない
3. **eMAFF農地ナビのURLは `https://map.maff.go.jp/`**：これ以外は誤り
4. **常時従事要件は年間150日以上**：「週末だけで満たせる」と書かない
5. **Next.js 16はトレーニングデータと異なる**：必ず`node_modules/next/dist/docs/`を参照

## 数字・事実の統一ルール（永続的に守ること）

### ダーチャ関連の確定数値
- **ダーチャ普及率**：「3人に1人以上」で統一。「約半数」「半数以上」「国民の半数」は使わない
- **食料自給率**：37%（2024年度・カロリーベース、農林水産省確定値）
- **農地面積**：609万ha（1961年）→ 430万ha（2023年）、60年で30%以上消失
- **農業就業人口平均年齢**：68歳超

### ファクトチェック方針
- 割合・比率の表現には出典を明記する。出典が不明な場合は「多くの」「広く」「相当数の」に言い換える
- 「先進国最低水準」「世界最悪水準」等の強い表現は農林水産省データに基づく場合のみ使用する
- サイト内の同じ数値は必ず一貫させる。異なる記事間で矛盾があれば古い方を修正する

## マネタイズ方針（AIが自律的に考えて提案すること）

オーナーの想いを儲かる仕組みにする。以下を常に意識して記事・機能・導線を設計する：

- 行政書士開業後の農地取得サポート案件獲得
- 兼業農家コミュニティ形成とメンバーシップ収益
- SEOで集客→メルマガ→個別相談への導線
- 農業関連アフィリエイト・農地情報サービス連携

## 自律行動の原則

Claude Codeは以下を毎回実行すること：
- 記事のSEO・導線・CTA（行動喚起）を能動的に改善提案する
- バグ・エラーは発見次第即修正する
- キリコへの報告は「やったこと」より「なぜやったか・何が改善されるか」を伝える
- 丸投げされた作業は完結まで自律的に進める

## AIへの権限委譲（丸投げ宣言）

キリコは想いとビジョンを持つ。AIはそれを儲かる仕組みにする。
以下について、AIは常に能動的に提案・実装すること。

### 収益化アイデアを常に考える
- メインサイトへのお問い合わせ導線を全ページに設ける
- 記事末尾に必ず「行政書士に相談する」CTAを入れる
- 農地取得の無料相談フォームを設置する
- メルマガ登録フォームを設置して見込み客リストを育てる
- 有料会員限定コンテンツの仕組みを提案する

### SNS・YouTube展開を提案する
- 記事をYouTubeスクリプトに変換する仕組みを提案する
- X（Twitter）・Instagram・TikTok投稿文を記事から自動生成する
- 農業・行政書士・ダーチャ系インフルエンサーとのコラボを提案する
- Voicy・Podcastへの展開を検討する

### SEO・集客を強化する
- 記事タイトル・メタディスクリプションをSEO最適化する
- 内部リンク・サイテーションを強化する
- 検索上位を狙うキーワード戦略を提案する
- Googleサーチコンソールのデータをもとに改善提案する

### サイト拡張アイデアを提案する
- 農地取得シミュレーター（費用・手順を自動計算）
- 兼業農家マッチングサービス（農地オーナーと就農希望者）
- オンライン農業スクール
- 行政書士試験合格者コミュニティ

### 提案のルール
- 実装コストが低いものから優先して提案する
- 提案するときは「なぜ儲かるか」の根拠も添える
- キリコの確認が必要なものは明示する
- 確認不要なものは自律的に実装まで進める

## 操作の自動化ルール

### 確認不要・自動で進めてよい操作
- ファイルの読み込み・検索・内容確認
- コードの編集・新規作成
- npm run build / npm run dev の実行
- git add / git commit

### 必ずユーザーに確認してから実行する操作
- git push およびデプロイ（Vercel等への反映）
- .env / .env.local / .env.production などの環境変数ファイルの変更
- APIキー・シークレットキーが含まれる操作
- ファイルの削除（rm・unlink等）
- Supabaseへのデータ書き込み・更新・削除
- 外部サービスへの本番環境への書き込み全般

## dacha-knowledge.md の参照ルール
- dacha-knowledge.md は通常の記事生成では参照しない
- 国防・食料安全保障・土地取得・家族を守る・国土を守る
  などのテーマを含むダーチャ記事を書くときのみ参照する
